## 2024-05-18 - [Fix Unhandled TypeError in Admin Login Endpoint]
**Vulnerability:** A Denial of Service (DoS) and unhandled exception risk existed in `server/api/admin/login.post.ts`. The endpoint read the `password` property from the JSON body but did not validate its type. A malicious user could send an array or object (e.g., `{"password": []}`) which would pass truthiness checks but cause `crypto.createHash('sha256').update(password)` to throw a synchronous `TypeError [ERR_INVALID_ARG_TYPE]`, resulting in a 500 Internal Server Error instead of a 400 Bad Request.
**Learning:** Even when checking for presence (`if (!password)`) and length (`if (password.length > 1024)`), type confusion can occur if the input is parsed from JSON (where it can be an object or array). Methods like `crypto.update()` strictly require strings or buffers.
**Prevention:** Always validate that the expected scalar type (e.g., `typeof password === 'string'`) is matched before passing user input to strictly-typed Node.js built-ins. Use Zod schemas where possible, or add explicit type guards.

## 2025-03-05 - SSRF Protection for Web Push Endpoints
**Vulnerability:** The `WebSubscriptionSchema` for web push notifications accepts any valid URL up to 500 characters. Since the backend sends a POST request to this endpoint to push notifications, an attacker could provide a malicious internal URL.
**Learning:** The built-in `.url()` validation of Zod only ensures format correctness, not destination safety. Endpoints controlled by users and fetched by the server must be strictly validated against internal/private IP ranges and forced to use HTTPS to prevent SSRF and internal port scanning. Using DNS resolution to check the resolved IP is necessary to prevent DNS rebinding or custom domains pointing to local IPs.
**Prevention:** Use a `.refine` method on URL fields that will be requested by the server to enforce HTTPS. To fully protect against SSRF, the HTTP client making the request must either resolve the hostname and check the IP against a blocklist, or block local/private network ranges at the socket level. As a first layer of defense, block known internal hostnames and IP strings.
